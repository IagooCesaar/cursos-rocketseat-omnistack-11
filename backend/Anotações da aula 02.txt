// -----------------------------------
    Métodos HTTP: (para melhor identificação das funções )

    GET     => Buscar uma informação do back-end (padrão dos browsers)
    POST    => Criar uma informação no back-end  
    PUT     => Alterar uma informação no back-end
    DELETE  => Deletar uma informação no back-end


// -----------------------------------
   Tipos de parâmetros

   QUERY    => 
      • São parâmetros nomeados que enviamos na Rota (URL) 
      • Geralmente são utilizados para filtros, paginação, etc
      
      Ex: http://localhost:3333/users?matricula=390
      
      obter acesso em : request.query.params
      const { matricula } = request.query.params;

   ROUTE    =>
      • São parâmetros utilizados para identificar recursos

      Ex: http://localhost:3333/users/390

      obter acesso em : request.params
      app.get('/users/:matricula', (request, response) => {
         const params = request.params;
      })
      
   REQUEST BODY =>
      • É o corpo da requisição e é utilizado geralmente para criar ou alterar recursos

      obter acesso em: request.body;

   HEADER =>
      • Geralmente utilizado para enviar dados do contexto da aplicação
         como: usuário autenticado, localização (idioma) 
      Ex: identificação do usuário logado

      obter acesso em : resquest.headers
